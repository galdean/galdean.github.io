Figma UI/UX Design Project

A class project in which I created a look-and-feel-prototype using Figma. The goal was to create a site that teaches a very specific subject to a chosen target audience. The focus was on going through the user-experience design process (defining user, user goals, wireframes, mood boards, mock-ups, etc.) to see how a user would interact with the site rather than assuming users would know exactly how to use the site and in order to catch any overlooked issues.

Paper Sales Log Using Autocomplete Widget

This site is for a fictional paper company to log paper sales, see previous sales, and delete sales. Implemented in HTML, CSS, Bootstrap, JavaScript, and jQuery without a back-end. The client box had jQuery autocomplete populated with a few names. New names entered by the user are added to the autocomplete list and appear as suggestions the next time the user clicks on the name entry box. Error checking and warnings are included when submitting sales. Cursor bounces back and clears data.here.

Hackathon Database (React Back-end and Frontend Data manipulation)

This site is intended for users interested in finding hackathons and information about them. Using, Flask, HTML, CSS, jQuery, and Bootstrap, this webpage allows users to interact with data by creating new data, viewing data, and searching the data. I used a Flask server in which I created a database which holds researched info on hackathons in the United States. New data created by the user would be added to the server, so the entry would still appear even if the page was refreshed. The search function finds key words in the title or summary and highlights the keywords in each of the results on the results page. The main page displays the newest 30 entries. Error checking as well as informing the user at every step that their actions succeeded, failed are included as part of the UI/UX focus. Watch the video to see site work plus more details.

Drag & Drop to edit a Party Planning Committe

If you click on this card to see the code in action, note that you need to double click on a person to grab them then you need to drop the name at the heading of the group you wish to place the person in. This page used jQuery Draggable and Droppable events to implement an interface where a user can drag names from non-party-planning-committee aka non-ppc to PPC and vice versa. UI/UX focused on cues to show elements are clickable and draggable by changing colors and shades.

Post a Tweet

Using HTML, JavaScript, and jQuery, this site allows users to post a tweet that will appear at the top of a list of older tweets. The focus of this project was to provide feedback to the users such as changing the number of characters left to write, using red and blocking the use of the post button when character count was exceeded, showing the new tweet in correct order, once posted: the old entry disappears but the cursor returns, etc.
